        <iframe src="www.exemplo.com" frameborder="0" width="500" height="500"></iframe> "in-line"

    4- Sobre a barra de rolagem na lateral, podemos por meio do parâmetro "scroling", alterar seu comportamento em relação ao "iframe:", e para isso usamos os seguintes valores:
            {auto} = Valor padrão, cria barra de rolagem na lateral se o conteúdo exceder o tamanho do frame.
            {yes}  = Valor para dizer que mesmo se o Conteúdo for maior que o frame, então mostra a barra.
            {no}   = Valor que não permite usar barra de rolagem mesmo que o conteúdo seja exceda o frame.
                OBS sobre {no}, em alguns dispositivos, o Google chrome permite, e em outros não, então fique sabendo que isso depende muito de navegador para navegador.
            
        + sobre:
           $ Visão geral criada por IA

            O scrolling no iframe é um atributo que permite controlar a visibilidade da barra de rolagem dentro do elemento <iframe>. 
            Os valores possíveis para o atributo são "yes", "no" ou "auto". Com "yes", a barra de rolagem é habilitada, com "no" é desabilitada, e com "auto" a barra de rolagem é exibida automaticamente quando necessário. 
            O atributo de scrolling é importante para garantir que o conteúdo dentro do iframe seja apresentado conforme o pretendido, melhorando a experiência do usuário na página. 
            O iFrame é um código em HTML que permite exibir um elemento dentro de outro. Ele é usado para incorporar documentos, vídeos e mídia interativa em uma página.

    Iniciamos esta aula criando o arquivo "iframe03".

NEsta aula é ensinado, que é possível adcionar conteúdos de navegação dentro de iframes.

    Os links de ligação para serem exibidos dentro de um <iframe> pecisa de um parâmero chamado {target=""}, e o valor deve ser o name do "objeto/elemento" iframe em questão.
        EX:
        "<ul>
            <li><a href="pag001.html" target="frame_name">Primeira página</a></li>
            <li><a href="pag002.html" target="frame_name">Segunda Página</a></li>
            <li><a href="pag003.html" target="frame_name">Terceira página</a></li>
        </ul>"

        "<iframe class="iframe003" id="tela"  name="frame_name" src="">"
            <p>Mensagem de incompatibilidade de navegador!</p>
        "</iframe>"

    Para adcionar um conteúdo para ficar estatíco(Tipo uma capa de ínicio) para que não fique um iframe vazio assim que carregar a página, use o parâmetro "src=", e entre aspas ("") adcione o caminho do aquivo que queres exibir.
        EX:
            "<iframe class="iframe003" id="tela"  name="frame_name" src="./imagens/capa-iframe.png">"
                <p>Mensagem de incompatibilidade de navegador!</p>
            "</iframe>"
    LEMBRE:
        O foco para o parâmetro  (src="") { é assunto para próxima aula }.

    Uma coisa que foi citado na aula anterior é a questão de que outros sites podem usar seu site, se for adcionado iframe para eles, e por meio do "iframe" permitir ataques de captura de informações de usuários; UMA dAS coisas que se pode fazer para resolver esse problema é {bloquear isso}; 
        AGORA criamos o arquivo "iframe004.html" para nele adicionarmos o formulário no "iframe" para entendermos sobre com bloquear acessos de terceiros.
    
    NOTE que ao clicarmos no botão de enviar no formulário dentro do iframe, ele envia os dados; 
        1- AGORA para bloquear, usamos a configuração por meio do parâmetro "sandbox" com o valor = "sandbox".
        $ Quando colocamos essa configuração, o iframe irá bloquear todo e qualquer acesso que outros sites que estejam dentro do Iframe possar tentar fazer; Como no exemplo o cadastro de pessoas.
        A configuração "sandbox" liga o máximo de segurança possível em relação ao iframe.
    { Quando usamos essa configuração no iframe, é como se uma proteção maior estivesse sendo adicionada a nosso site, e evitando que dados sejam capturados por ataques de captura de algum tipo de dado.}
        2- OUTRO parâmetro é o "referrerpolicy" que é a politica de referência, que tem por padrão o valor = "no-referrer-when-downgrade.", e para limitar carregamentos dentro do iframe, podemos usar o valro = "no-referrer".
OBS:
    Sites capturam dados de navegação para com base nesses dados, comece a enviar e mostrar propagandas, e ao usar o parâmetro e o valor "no-referrer" , então dizemos que tudo o que estiver carregado dentro do iframe (o conteúdo), "NÃO VAI coletar DADOS ou qualquer tipo de informação do usuário".

* MAS e se quisermos que no iframe possa ser coletado algum tipo de dado? Como no exemplo o envio dos dados de um fomulário confiável? COmo podemos desligar algumas coisas e não usar toda segurança possível?
    "COMO valor para o parâmetro" {sandbox}; POdemos usar:
        1- "allow-same-origin" Para páginas de mesma origem(servidor) que nosso site = Permitir Conteúdos de Mesma Origem.
        2- "allow-forms allow-scripts" = Permitir Formulários e Scripts "USamos assim porque geralmente formulários usa de scripts para manipular dados."
            $   OBS: PODEMOS usar mais valores configurativos, para isso entre aspas "" do sandbox, tecle Ctrl+Space e veja mais opções.

e sobrenome nos campos de dados e enviamos esses dados; os mesmo aparecem na barra de endereços(URL) do navegador e isso é padrão no envio de formulários e pode nos levar a pensar que seja uma falta de segurança.
    MAs pode ser que seja uma falta de segurança, porém pode ser algo impensado por parte do programador, já
que se pode desativar por meio de metodos(parâmetros) se queremos que não apareça os dados na URl.
    Outra coisa que podemos entender é que formulários usam por padrão esse parâmetro de mostra na URL o

programação; Porém existem dois metódos simples para envio de formulário.
    1º-  É esse padrão que não precisamos definir algo dentro da tag <form>, e ele se chama metódo (GET).
PODEMOS ainda por meio do atributo "method" definido da seguinte dorma: <form method=""> e entre aspas duplas por meio do CTRL ver os parâmetros que são o padrão {GET} , o {POST} e o {dialog}.

    2º- Esse é o metódo POST que faz com que esses dados não apareçam na URL da barra de endereços do
navegador.

        https://www.google.com/search?q=virus+e+bac%C3%A9ria&oq=virus+e+bac%C3%A9ria&aqs=chrome..69i57j0i13i512l9.4124j0j7&sourceid=chrome&ie=UTF-8

    Então podemos por causa do GET, pegar apenas a pesquisa que ficou no Endereço URL; Como no exemplo abaixo, que ao ser compartilhada, basta o usuário copiar e colar no campo de URL, que vai aparecer exatamente o que foi pesquisado, sem que o usuário precise digitar novamente a pesquisa.
        https://www.google.com/search?q=virus+e+bact%C3%A9ria+diferen%C3%A7a

* SAIBA QUE:
    1- Ao usar GET, é possível enviar apenas 3000Bytes, caso utrapasse os Bytes, então esse não é  melhor "method".
        1a- 3000Bytes é aproximadamente 3000letras; Então o formulário inteiro usando method GET, não podem possuir mais de 3K_letras. 
        1b- Não poderar usar para campos sensíveis, que utilizam senhas, cartões bancários etc; Não poderá usar para envio de fotos ou qualquer tipo de arquivo.
    2- Usamos POST para envio de dados sensíveis e para que os dados não apareça diretamente na URL; "Porém se a questão é dados bancários, então é bom usar {https} que é de suma importâcnia aprender no futuro breve e urgente, no que diz respeito a proteção de dados}".
        2a- Usamos para Posts com mais de 3K_letras.
        2b- Usamos para envio de arquivos.

*       SOBRE a propriedade "media=screnn e media=print" em CSS:
        As propriedades @media screen e @media print em CSS são usadas para definir estilos diferentes para cada tipo de mídia:
        @media screen: Define o CSS para telas
        @media print: Define o CSS para impressão 
        A regra @media é uma técnica de CSS3 que permite aplicar estilos diferentes para diferentes dispositivos ou tamanhos de tela. Ela funciona da seguinte forma:
        A regra @media verifica os tipos de dispositivos cobertos pelas media types e a capacidade deles
        A regra @media efetua consultas ao dispositivo que o usuário está usando para acessar o site
        A regra @media especifica o estilo desejado conforme as características do dispositivo consultado 
        A regra @media permite: 
        Aplicar estilos diferentes para diferentes dispositivos ou tamanhos de tela 
        Executar testes, como se a tela do usuário é maior que uma certa largura, ou uma certa resolução 
        Estilizar a página de acordo com as necessidades do usuário 

    "
        A proriedade media em CSS definem midia e determina quando as regras do CSS devem ser utilizadas. O valor da propriedade deve ser um query de midia.
        A regra @media é usada para aplicar estilos diferentes para diferentes dispositivos ou tamanhos de tela. Por exemplo, para definir um tamanho de tela de, no máximo, "700px" de largura, você usaria o parâmetro "max-width: 700px" na sua regra @media.

*   Falando sobre acelerômetro em dispositivos com "orientação de tela".
Visão geral criada por IA
    O acelerômetro é um sensor que mede a aceleração de um objeto, ou seja, a taxa de variação da velocidade. Nos celulares, ele é usado para detectar movimentos em todas as direções e interagir com aplicativos e a interface do aparelho. 
    O acelerômetro funciona da seguinte forma:
    Quando o dispositivo é acelerado, a massa interna se desloca.
    Esse deslocamento é convertido em sinais elétricos.
    O processador do dispositivo interpreta os sinais elétricos e traduz as variações de aceleração. 
    O acelerômetro é usado em diversos dispositivos, como: Celulares e tablets Android, Câmeras digitais, Notebooks, Airbags de carros, Tênis. 
    Nos celulares, o acelerômetro é usado para:
    Detectar a inclinação do aparelho 
    Mudar a interface do aparelho 
    Interagir com aplicativos de acordo com os movimentos 
    Ajustar a orientação do que é mostrado na tela, como em jogos
